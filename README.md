## 🌟 The Vision: Elite Pro 10/10 Architecture

The conventional gap between dense government data and the end citizen is massive. The CPI Platform bridges this divide with unprecedented scale, transparency, and localization. 

This **Elite 10/10 Architecture** introduces strict Jurisdictional Logic (Nation vs. State) and hyper-localized data processing for 1.4 Billion citizens across 28 States and 8 Union Territories.

### 🏛️ 1. Absolute Political Hierarchy
- **The Engine:** PostgreSQL-backed strict jurisdictional mapping.
- **The Rulers:** Seamlessly mapping 543 Lok Sabha MPs, 245 Rajya Sabha MPs, and thousands of State MLAs (Vidhan Sabha) directly to their respective State or National contexts.
- **The Magic:** A user from Tamil Nadu instantly sees their specific MLAs alongside National MPs, without any cross-state data bleed.

### 💰 2. Multi-Tiered Budget Intelligence
- **National vs. State Splits:** The engine distinguishes between the Union Budget (Defence, Highways) and 28 independent State Budgets (Agriculture, Local Education).
- **Visual Analytics:** Real-time visual comparison of a State's contribution to the Nation vs. its internal consumption using Recharts and D3.js interactive Treemaps.

### 📜 3. Hyper-Local Scheme Logic (Deterministic)
- **Zero Hallucination Routing:** A rule-based Engine that aggregates `Central Schemes (e.g., PM-KISAN)` + `Specific State Schemes (e.g., Rythu Bandhu in Telangana)`.
- **Intelligent Filtering:** Prevents a citizen in Gujarat from seeing a scheme exclusively applicable to Kerala.

### 📰 4. The Dual-Core Newspaper Engine
The platform recognizes two distinct modes of news delivery to prevent hallucinations while maximizing localized value:
- **Original Live Wire (Google RSS Integration):** The real-time, minute-by-minute news feed is strictly sourced from verified **Google News RSS feeds**. It pulls authentic regional news for all 28 states, guaranteeing the original source of truth.
- **The E-Paper Edition (Gemini AI):** The 11-page downloadable PDF newspaper (E-Paper) is generated by Gemini Pro. It synthesizes purely factual, unbiased data from our Parliament and Budget engines, translating it into an elite, multi-lingual "Morning Edition" for every state natively.

## ⚡ Quick Start Guide: Elite Edition

### Requirements
- PostgreSQL Server Running
- Node.js 20+
- Python 3.10+
- pnpm package manager

### Terminal 1: Core Backend Service ⚙️
```bash
cd backend
python -m pip install -r requirements.txt
python -m pip install psycopg2-binary asyncpg alembic

# Initialize PostgreSQL Schema
alembic upgrade head

# Inject Massive State/National Hierarchy Data
python seed_real_data.py
python seed_members.py

# Launch the High-Performance ASGI Server
uvicorn app.main:app --reload --port 8000
```

### Terminal 2: NLP Backend Service 🧠
```bash
cd backend_nlp
pip install -r requirements.txt
python server/manage.py runserver 8001
```

### Terminal 3: Frontend Citadel 🌏
```bash
cd frontend-citizen
pnpm install
pnpm build
pnpm run dev
```

##   Performance Optimization (New)

### 1. Smart vendor chunk splitting (in `frontend-citizen/vite.config.ts`)
- `react-vendor` for React + React DOM + React Query
- `network-vendor` for Axios
- `charts-vendor` for chart libs (recharts/chart.js/d3)
- `map-vendor` for Leaflet + React-Leaflet
- fallback `vendor` for other third-party modules

### 2. Route-level code splitting (in `frontend-citizen/src/App.tsx`)
- React.lazy + Suspense used for all page routes
- Only the active route loads initial JS

### 3. Feature-level lazy imports (e.g., for heavy NLP charts)
- Use `const SomeChart = lazy(() => import('./components/SomeChart'));`
- Wrap with `Suspense` fallback

### 4. Bundle analysis guide
- Install:
  - `cd frontend-citizen && pnpm add -D rollup-plugin-visualizer`
- Update `vite.config.ts`:
  - `import { visualizer } from 'rollup-plugin-visualizer'`
  - `plugins: [react(), visualizer({ open: true, filename: 'stats.html' })]`
- Run build & inspect:
  - `pnpm build`, open `stats.html`

> Note: `chunkSizeWarningLimit` is useful but only suppresses warnings; real performance improvements come from splitting.

##  📡 Elite API Capabilities

The backend provides highly segregated endpoints based on jurisdictional context:
- `GET /api/members/lok-sabha` - Fetch National MPs
- `GET /api/members/mlas?state=MH` - Fetch Vidhan Sabha MLAs exclusively for Maharashtra
- `GET /api/budget/compare?state=TN` - Compare Tamil Nadu's State budget dynamically.

**Documentation:** `http://localhost:8000/docs`
